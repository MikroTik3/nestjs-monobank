У цьому розділі ми розглянемо, як отримати поточний статус регулярної підписки. Ми покажемо, як викликати API Monobank та обробити відповідь від сервісу.

### Виклик методу для отримання статусу підписки

Для отримання інформації про підписку достатньо передати її ідентифікатор (`subscriptionId`) у метод `getStatus` сервісу `MonobankService`.

```typescript
import { Injectable } from '@nestjs/common';
import { MonobankService } from 'nestjs-monobank';

@Injectable()
export class SubscriptionService {
     constructor(
          private readonly monobankService: MonobankService
     ) {}

     async getSubscriptionStatus() {
          const status =
               await this.monobankService.subscription.getStatus(
                    'sub_9ZgpZVsl3'
               );

          return status;
     }
}
```

### Обробка відповіді від API

У відповіді API повертає повну інформацію про стан підписки, статистику платежів та інформацію про гаманець, через який виконуються списання.

```json
{
     "subscriptionId": "s2_AbrCdXyZ13",
     "status": "active",
     "startDate": "2024-06-01T14:15:22Z",
     "endDate": "2024-06-20T09:30:15Z",
     "amount": 4200,
     "ccy": 980,
     "interval": "1m",
     "nextChargeDate": "2024-07-01T14:15:22Z",
     "cancellationDesc": "Скасовано за запитом користувача",
     "summary": {
          "totalPaid": 3,
          "totalFailed": 1
     },
     "walletData": {
          "cardToken": "67XZtXdR4NpKU3",
          "walletId": "c1376a611e17b059aeaf96b73258da9c",
          "status": "created",
          "failureDescription": "Недостатньо коштів на картці"
     }
}
```

Поле `status` містить поточний стан підписки. Об'єкт `summary` повертає статистику успішних та невдалих списань, а `walletData` містить інформацію про гаманець, який використовується для регулярних платежів, включаючи його статус та можливу причину помилки, якщо списання не вдалося.

За допомогою цього методу можна перевірити, чи є підписка активною, коли відбудеться наступне списання, а також отримати додаткову інформацію про історію роботи підписки та прив'язаний платіжний гаманець.
