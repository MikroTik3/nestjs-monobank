---
title: Керування підпискою
description: Кроки для скасування регулярної підписки
---

У цьому розділі ми розглянемо, як керувати регулярною підпискою. Наразі API Monobank підтримує скасування підписки. Ми покажемо, як підготувати дані, викликати API Monobank та що відбувається після виконання запиту.

### Підготовка даних для скасування підписки

Для скасування підписки необхідно підготувати об'єкт `EditSubscriptionRequest`, який містить ідентифікатор підписки та дію `cancel`. За потреби також можна вказати суму повернення коштів.

```typescript
import { type EditSubscriptionRequest } from 'nestjs-monobank';

const subscriptionData: EditSubscriptionRequest = {
     subscriptionId: 'sub_9ZgpZVsl3',
     action: 'cancel',

     // Необов'язково
     refundAmount: 19900
};
```

### Виклик методу для скасування підписки

Після підготовки даних можна скористатися методом `editSubscription` із сервісу `MonobankService`.

```typescript
import { Injectable } from '@nestjs/common';
import {
     type EditSubscriptionRequest,
     MonobankService
} from 'nestjs-monobank';

@Injectable()
export class SubscriptionService {
     constructor(
          private readonly monobankService: MonobankService
     ) {}

     async cancelSubscription() {
          const data: EditSubscriptionRequest = {
               subscriptionId: 'sub_9ZgpZVsl3',
               action: 'cancel',

               // Необов'язково
               refundAmount: 19900
          };

          await this.monobankService.subscription.editSubscription(data);
     }
}
```

### Обробка відповіді від API

Метод `editSubscription` не повертає даних. Якщо запит було виконано успішно, підписка буде скасована.

Якщо передано поле `refundAmount`, Monobank також виконає повернення зазначеної суми на картку користувача. Якщо поле не вказано, підписка буде лише скасована без повернення коштів.

Після скасування підписки Monobank більше не виконуватиме автоматичні регулярні списання за цією підпискою.
