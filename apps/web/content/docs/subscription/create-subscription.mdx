У цьому розділі ми розглянемо, як створити регулярну підписку (Recurring Payment). Ми покажемо, як підготувати дані для створення підписки, викликати API Monobank та обробити відповідь від сервісу.

### Підготовка даних для створення підписки

Для створення підписки необхідно підготувати об'єкт `CreateSubscriptionRequest`, який містить інформацію про суму платежу, періодичність списання та URL для webhook-подій.

```typescript
import { type CreateSubscriptionRequest } from 'nestjs-monobank';

const subscriptionData: CreateSubscriptionRequest = {
     amount: 19900,
     ccy: 980, // UAH
     interval: '1m',
     redirectUrl: 'https://example.com/payment/success',
     webhookUrls: {
          chargeUrl: 'https://example.com/api/v1/webhooks/monobank/charge',
          statusUrl: 'https://example.com/api/v1/webhooks/monobank/status'
     },
     validity: 3600 * 24 * 30
};
```

### Виклик методу для створення підписки

Після підготовки даних можна скористатися методом `create` із сервісу `MonobankService`. У відповідь буде отримано посилання, на яке необхідно перенаправити користувача для підтвердження регулярних списань.

```typescript
import { Injectable } from '@nestjs/common';
import {
     type CreateSubscriptionRequest,
     MonobankService
} from 'nestjs-monobank';

@Injectable()
export class SubscriptionService {
     constructor(
          private readonly monobankService: MonobankService
     ) {}

     async createSubscription() {
          const data: CreateSubscriptionRequest = {
               amount: 19900,
               ccy: 980,
               interval: '1m',
               redirectUrl: 'https://example.com/payment/success',
               webhookUrls: {
                    chargeUrl:
                         'https://example.com/api/v1/webhooks/monobank/charge',
                    statusUrl:
                         'https://example.com/api/v1/webhooks/monobank/status'
               },
               validity: 3600 * 24 * 30
          };

          const subscription =
               await this.monobankService.subscription.create(data);

          return subscription;
     }
}
```

### Обробка відповіді від API

У відповідь API повертає ідентифікатор підписки (`subscriptionId`) та посилання (`pageUrl`), на яке необхідно перенаправити користувача для підтвердження регулярних платежів.

```json
{
     "subscriptionId": "sub_9ZgpZVsl3",
     "pageUrl": "https://pay.mbnk.biz/subscription/sub_9ZgpZVsl3"
}
```

Після успішного створення підписки користувача необхідно перенаправити на `pageUrl`. Після підтвердження картки Monobank почне автоматично виконувати регулярні списання відповідно до значення поля `interval`.

Подальшу інформацію про стан підписки та успішні або невдалі списання можна отримувати через webhook-и, вказані в `webhookUrls`.
